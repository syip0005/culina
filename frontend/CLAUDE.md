# Culina Frontend

## Project Structure

- `src/routes/` — TanStack Router file-based routes (auto-generates `routeTree.gen.ts`)
- `src/routes/__root.tsx` — root layout, wraps app in AuthProvider
- `src/routes/_authenticated.tsx` — layout guard that redirects unauthenticated users to `/login`
- `src/routes/_authenticated/` — protected pages (home `/`, settings `/settings`)
- `src/components/` — shared React components
- `src/utils/` — utility hooks and helpers
- `src/api.ts` — typed API client, all backend calls go through `request<T>()`
- `src/auth.tsx` — AuthProvider context + `useAuth()` hook (Supabase session + backend user)
- `src/types.ts` — TypeScript interfaces mirroring backend schemas
- `src/supabase.ts` — Supabase client singleton
- `src/index.css` — all CSS (brutalist black-and-white design system, no CSS modules)

## Commands

- `npm run dev` — start Vite dev server on port 3000
- `npm run build` — TypeScript check + Vite production build
- `npx tsc -b` — TypeScript type check only
- `npx vite build` — production build (also triggers route tree generation)

## Architecture

### Routing
- TanStack Router with file-based routing and auto code-splitting
- Vite plugin `TanStackRouterVite` generates `src/routeTree.gen.ts` — do not edit this file
- `_authenticated` prefix = layout route with auth guard (not a URL segment)

### Auth Flow
- Supabase OAuth (Google, GitHub) handles login/token refresh
- `AuthProvider` subscribes to `onAuthStateChange`, fetches backend user via `GET /auth/me`
- `useAuth()` provides `{ session, user, loading, signOut }`
- API client reads token from `supabase.auth.getSession()` for every request

### API Client (`api.ts`)
- `request<T>(method, path, body?)` — adds Bearer token, handles JSON, throws on non-OK
- All backend endpoints have named exports (e.g. `getMe`, `listMeals`, `searchEntries`, `lookup`)
- Backend URL from `VITE_BACKEND_URL` env var

### Main Screen Flow
- Home page fetches daily summary + today's meals in parallel
- Meals mapped to 4 sections: breakfast, lunch, dinner, snacks
- NutritionEntry data fetched per meal item to display names/macros
- "Add Item" opens full-screen overlay with search + AI lookup

### Add Item Flow
- `AddItemPanel` — search overlay with debounced keyword (5) + semantic (8) parallel search, deduped to 8 max results
- "Find New Items" opens `LookupView` as second overlay on top
- Lazy meal creation: meal is created on first item add if it doesn't exist yet

### AI Lookup (`LookupView`)
- Multi-turn conversation via `POST /lookup/` with `conversation_id`
- Handles `FollowUpResponse` (question + clickable buttons) and `NutritionResultResponse` (items with ADD)
- Adding a lookup result: creates NutritionEntry, then MealItem with chosen quantity
- Text input always visible for continued conversation

## Design System
- Brutalist black-and-white: `--color-bg: #fff`, `--color-fg: #000`, `--border: 2px solid #000`
- No border-radius, no shadows, uppercase bold typography
- All styles in `src/index.css` using CSS custom properties
- System font stack: `system-ui, -apple-system, 'Segoe UI', Roboto, Helvetica, Arial, sans-serif`

## Key Conventions
- React 19 + TypeScript strict mode + Vite 8
- No state management library — React useState/useEffect/useContext
- All CSS in `index.css`, class-based (no CSS-in-JS, no CSS modules)
- Types mirror backend Pydantic schemas exactly (see `types.ts`)
- `verbatimModuleSyntax` enabled — use `import type` for type-only imports

## Environment Variables
- `VITE_SUPABASE_URL` — Supabase project URL
- `VITE_SUPABASE_ANON_KEY` — Supabase anonymous/publishable key
- `VITE_BACKEND_URL` — backend API base URL (default `http://localhost:8000`)

## Backend API Surface
- `GET /auth/me` — current user
- `GET/PATCH /users/me/settings` — user settings CRUD
- `GET /meals/` — list meals (query params: `eaten_after`, `eaten_before`, `meal_type`)
- `POST /meals/` — create meal
- `POST /meals/{id}/items` — add item to meal
- `DELETE /meals/{id}/items/{item_id}` — remove item from meal
- `POST /nutrition-entries/search` — search entries (`query`, `mode`, `limit`)
- `POST /nutrition-entries/` — create entry
- `GET /nutrition-entries/{id}` — get single entry
- `POST /lookup/` — AI nutrition lookup (multi-turn)
- `GET /summary/daily?date=` — daily macro summary
